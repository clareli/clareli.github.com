Markdown 简介

Markdown 是一种用来写作的轻量级「标记语言」，它用简洁的语法代替排版，发布格式随心所欲，你可以导出 HTML 文件，也可以导出 PDF 文件。目前主流网站博客（git、stackoverflow、简书）等都支持 Markdown 语法进行编辑输入。

" Markdown 格式化语法设计的目的就是为了易读 "-- Markdown 的创始人 John Gruber

Markdown 特点

  • 学习门槛低 (献给写作者的 Markdown 新手指南) ，极易上手，可以专注于文字内容而不是排版格式，写作效率较高，Markdown 写作浅谈
  • 可以很方便的生成目录，一目了然
  • 完美呈现代码，支持按照不同语言进行高亮显示，并支持输出复杂数学公式、流程图等，非常适合文档记录
  • 轻松发布成 HTML、PDF 格式文件，HTML 文件配合 css 可以完美适配各种终端，浏览查阅效果极佳
  • 纯文本内容，兼容所有的文本编辑器和字处理软件，可以随时修改

Markdown 书写工具

在线工具 stackedit

优点

  • 开源，免费，搭建在 GitHub page 上，源码寄存在 GitHub。支持 Markdown Extra，也就是全部高级语法功能，界面美观友好，导出 HTML 推荐使用 Export to disk > Using template 带样式直接输出，可以适配各种终端

缺点

  • 目前只有在线版本

sublime结合插件使用

优点

  • 只需安装相应插件就可实现语法高亮、文件预览、导出 HTML 等功能，推荐 markdown preview

缺点

  • 无法实时预览，只能在浏览器中查看，导出的 HTML 代码有冗余

客户端工具 MarkdownPad 2（官方软件）

优点

  • 客户端功能强大，界面友好，支持实时预览，导出文件方便

缺点

  • 需要购买 pro 版本才能支持 table 标签输出等高级语法

客户端工具 Cmd Markdown

优点

  • 国人开发，用详细的中文文档，既有在线版也有客户端版本，功能强大，界面友好，支持实时预览，导出文件方便

缺点

  • 需要注册才可使用，导出的 HTML 有大量冗余，不够简洁，部分高级功能需要购买会员资格才可以使用

轻松写文档才是使用 Markdown 的目的

  • 开发中，除了语言上的沟通，有时文档也是很有帮助的，但打开word是不是就不想写了，因为还要调整排版和格式，贴了代码也没法看，太TM麻烦了，烦！！所以连文档都省了，囧rz
  • 代码注释非常重要，但毕竟只能起到简单说明的作用，时间久了容易遗忘细节
  • 个人建议用 Markdown 写个文档补充，记录说明一下开发中较为复杂的逻辑或遇到的坑等等，对于自己或其他同学进行修改维护也是很有帮助的：）
